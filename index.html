What This Tier Builds

The simplest possible Jenkins job: one stage that tells Testsigma to start your Test Plan. That's it.

Does Jenkins wait for the Test Plan to finish? No. Testsigma's trigger API replies immediately with a Run ID — it doesn't hold the connection open until the plan finishes. The moment that reply arrives, this job's one stage succeeds, and the build is done. Testsigma keeps running your Test Plan completely on its own after that, independent of Jenkins. If you want the build to wait for a real pass/fail result, or to be able to abort the run from Jenkins, use Tier 2 in the sidebar.

Create the Scripts Folder

This tier needs one script: trigger-run.sh (Mac) or trigger-run.ps1 (Windows). It lives on the same machine as Jenkins — Jenkins runs Jenkinsfiles straight from its own job configuration, so unlike a repo-based tool, there's no separate checkout step; scripts just need to sit somewhere Jenkins can call them by path.

14

Create the folder:

mkdir -p /Users/$(whoami)/jenkins-ts-scripts
15

Save as trigger-run.sh in that folder:

#!/bin/bash
# -------------------------------------------------------------
# trigger-run.sh
# Triggers a Testsigma execution and saves the Run ID to a
# fixed path in /tmp so the abort hook can always find it,
# regardless of workspace state.
#
# Required environment variables (set in the Jenkinsfile):
#   TS_API_TOKEN  -- Testsigma API key (Jenkins Credential TS_API_KEY)
#   TS_PLAN_ID    -- Execution ID from Testsigma CI/CD Integrations tab
#   APP_VERSION   -- passed as buildNo for traceability in Testsigma UI
#   RUN_ID_FILE   -- fixed path set in the Jenkinsfile environment block
# -------------------------------------------------------------

set -euo pipefail

if [[ -z "${TS_PLAN_ID:-}" || -z "${TS_API_TOKEN:-}" ]]; then
  echo "[ERROR] TS_PLAN_ID and TS_API_TOKEN must be set."
  exit 1
fi

echo "[INFO] Triggering plan ID: ${TS_PLAN_ID} | Version: ${APP_VERSION:-unset}"

TMPFILE=$(mktemp)
HTTP_CODE=$(curl -s -o "$TMPFILE" -w "%{http_code}" \
  -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -H "Authorization: Bearer ${TS_API_TOKEN}" \
  "https://app.testsigma.com/api/v1/execution_results" \
  -d "{\"executionId\": \"${TS_PLAN_ID}\", \"buildNo\": \"${APP_VERSION:-manual}\"}")

BODY=$(cat "$TMPFILE")
rm -f "$TMPFILE"

if [[ "$HTTP_CODE" != "200" && "$HTTP_CODE" != "201" ]]; then
  echo "[ERROR] Testsigma returned HTTP ${HTTP_CODE}"
  echo "$BODY"
  exit 1
fi

RUN_ID=$(echo "$BODY" | python3 -c "import sys,json; print(json.load(sys.stdin)['id'])")

if [[ -z "$RUN_ID" || "$RUN_ID" == "null" ]]; then
  echo "[ERROR] Could not read Run ID from response."
  echo "$BODY"
  exit 1
fi

echo "$RUN_ID" > "${RUN_ID_FILE:-/tmp/ts_run_id_manual-run}"
echo "[OK] Execution started. Run ID: ${RUN_ID}"
echo "[INFO] Run ID saved to: ${RUN_ID_FILE:-/tmp/ts_run_id_manual-run}"

$ErrorActionPreference = "Stop"

if (-not $env:TS_PLAN_ID -or -not $env:TS_API_TOKEN) {
  Write-Host "[ERROR] TS_PLAN_ID and TS_API_TOKEN must be set."
  exit 1
}

Write-Host "[INFO] Triggering plan ID: $($env:TS_PLAN_ID) | Version: $($env:APP_VERSION)"

try {
  $body = @{ executionId = $env:TS_PLAN_ID; buildNo = $(if ($env:APP_VERSION) { $env:APP_VERSION } else { "manual" }) } | ConvertTo-Json
  $response = Invoke-RestMethod -Uri "https://app.testsigma.com/api/v1/execution_results" `
    -Method Post `
    -Headers @{ "Authorization" = "Bearer $env:TS_API_TOKEN"; "Accept" = "application/json" } `
    -ContentType "application/json" `
    -Body $body
} catch {
  Write-Host "[ERROR] Testsigma request failed: $($_.Exception.Message)"
  exit 1
}

$runId = $response.id
if (-not $runId) {
  Write-Host "[ERROR] Could not read Run ID from response."
  exit 1
}

$runIdFile = if ($env:RUN_ID_FILE) { $env:RUN_ID_FILE } else { "$env:TEMP\ts_run_id_manual-run.txt" }
Set-Content -Path $runIdFile -Value $runId -NoNewline
Write-Host "[OK] Execution started. Run ID: $runId"
Write-Host "[INFO] Run ID saved to: $runIdFile"

What This Tier Builds

Everything in Tier 1, plus: the build waits until the Test Plan actually finishes, and clicking Abort in Jenkins actually stops the Testsigma run instead of abandoning it.

Why doesn't Abort already do this by default? Jenkins has no built-in awareness of Testsigma. Clicking Abort only stops Jenkins' own build — it sends nothing to Testsigma, so the Test Plan keeps running and keeps using your Testsigma license. Making Abort actually work needs two things: a stage that's still running when you click Abort (that's the Wait stage below — Tier 1 has nothing to abort because it already finished), and a post { aborted { ... } } block that explicitly tells Testsigma to stop. Jenkins runs this block automatically whenever a build is aborted — no extra timeout settings needed to make it fire.

Save as poll-status.sh, same folder — polls on terminal status, with the 4-hour failsafe and an independent circuit breaker for a dead API connection:

#!/bin/bash
# -------------------------------------------------------------
# poll-status.sh
# Polls Testsigma until the run reaches a terminal status, or
# (optionally) a safety-cap timeout elapses -- whichever comes
# first. Toggle the cap via MAX_WAIT_MINUTES without editing this file.
# Even when the cap is disabled ("unbounded"/"none"/0), an absolute
# ceiling (ABSOLUTE_MAX_WAIT_MINUTES, default 1440 = 24h) still applies,
# so this script can never loop truly forever. It also fails safe if
# Testsigma ever returns a result value outside the known terminal
# list on two consecutive polls, rather than hanging on an unrecognized
# status until a timeout eventually fires.
#
# Required: TS_API_TOKEN, RUN_ID_FILE
# Optional: MAX_WAIT_MINUTES (default: 240 = 4h; 0/"unbounded"/"none" disables
#             the normal cap in favor of the absolute ceiling below),
#           ABSOLUTE_MAX_WAIT_MINUTES (default: 1440 = 24h; last-resort
#             ceiling that applies even when MAX_WAIT_MINUTES is unbounded),
#           POLL_INTERVAL_SECONDS (default: 30),
#           MAX_CONSECUTIVE_HTTP_FAILURES (default: 10)
# -------------------------------------------------------------

set -uo pipefail
# NOTE: 'set -e' intentionally omitted -- one failed curl inside the loop
# must not kill the whole script; every HTTP call is checked explicitly.

RUN_ID_FILE="${RUN_ID_FILE:-/tmp/ts_run_id_manual-run}"
POLL_INTERVAL="${POLL_INTERVAL_SECONDS:-30}"
MAX_HTTP_FAILURES="${MAX_CONSECUTIVE_HTTP_FAILURES:-10}"
ABSOLUTE_MAX_MINUTES="${ABSOLUTE_MAX_WAIT_MINUTES:-1440}"

if [[ -z "${TS_API_TOKEN:-}" ]]; then
  echo "[ERROR] TS_API_TOKEN is not set."; exit 1
fi
if [[ ! -f "$RUN_ID_FILE" ]]; then
  echo "[ERROR] Run ID file not found at ${RUN_ID_FILE}. Did trigger-run.sh complete successfully?"; exit 1
fi
RUN_ID=$(cat "$RUN_ID_FILE")

RAW_MAX_WAIT="${MAX_WAIT_MINUTES:-240}"
RAW_MAX_WAIT_LC=$(echo "$RAW_MAX_WAIT" | tr '[:upper:]' '[:lower:]')
if [[ -z "$RAW_MAX_WAIT" || "$RAW_MAX_WAIT_LC" == "0" || "$RAW_MAX_WAIT_LC" == "unbounded" || "$RAW_MAX_WAIT_LC" == "none" ]]; then
  CAP_MODE="absolute-ceiling"; MAX_SECONDS=$(( ABSOLUTE_MAX_MINUTES * 60 )); DISPLAY_MAX_MINUTES="$ABSOLUTE_MAX_MINUTES"
  echo "[INFO] MAX_WAIT_MINUTES='${RAW_MAX_WAIT}' -> normal safety cap disabled. Polling until terminal status, backstopped by an absolute ceiling of ${ABSOLUTE_MAX_MINUTES} minute(s) (override via ABSOLUTE_MAX_WAIT_MINUTES)."
else
  if ! [[ "$RAW_MAX_WAIT" =~ ^[0-9]+$ ]]; then
    echo "[ERROR] MAX_WAIT_MINUTES must be a non-negative integer, 'unbounded', or 'none'. Got: '${RAW_MAX_WAIT}'"; exit 1
  fi
  CAP_MODE="normal"; MAX_SECONDS=$(( RAW_MAX_WAIT * 60 )); DISPLAY_MAX_MINUTES="$RAW_MAX_WAIT"
  echo "[INFO] MAX_WAIT_MINUTES='${RAW_MAX_WAIT}' -> safety cap ENABLED at ${MAX_SECONDS}s."
fi

# Statuses treated as "still in progress" -- never counted toward the
# unrecognized-status fail-safe below. UNKNOWN is this script's own
# placeholder when Testsigma's response has no 'result' field at all.
IN_PROGRESS_STATUSES=" running pending queued in_progress inprogress scheduled initiated started waiting not_started notstarted unknown "

ELAPSED=0
CONSECUTIVE_HTTP_FAILURES=0
LAST_UNRECOGNIZED_LC=""
echo "[INFO] Polling Testsigma for Run ID: ${RUN_ID}"

while true; do
  if [[ "$ELAPSED" -ge "$MAX_SECONDS" ]]; then
    if [[ "$CAP_MODE" == "absolute-ceiling" ]]; then
      echo "[TIMEOUT] Absolute ceiling of ${DISPLAY_MAX_MINUTES} minute(s) reached (MAX_WAIT_MINUTES was set to '${RAW_MAX_WAIT}'). Marking build FAILED."
    else
      echo "[TIMEOUT] Safety cap of ${DISPLAY_MAX_MINUTES} minute(s) reached. Marking build FAILED."
    fi
    echo "[TIMEOUT] Testsigma Run ID ${RUN_ID} is left RUNNING -- this script does not stop it on timeout, by design."
    exit 1
  fi

  TMPFILE=$(mktemp)
  HTTP_CODE=$(curl -s -o "$TMPFILE" -w "%{http_code}" \
    -X GET -H "Authorization: Bearer ${TS_API_TOKEN}" -H "Accept: application/json" \
    "https://app.testsigma.com/api/v1/execution_results/${RUN_ID}")
  BODY=$(cat "$TMPFILE"); rm -f "$TMPFILE"

  if [[ "$HTTP_CODE" != "200" ]]; then
    CONSECUTIVE_HTTP_FAILURES=$(( CONSECUTIVE_HTTP_FAILURES + 1 ))
    echo "[WARN] Poll failed HTTP ${HTTP_CODE} (${CONSECUTIVE_HTTP_FAILURES}/${MAX_HTTP_FAILURES}). Body: ${BODY}"
    if [[ "$CONSECUTIVE_HTTP_FAILURES" -ge "$MAX_HTTP_FAILURES" ]]; then
      echo "[ERROR] Testsigma API unreachable ${MAX_HTTP_FAILURES} polls in a row. Failing the build."; exit 1
    fi
    sleep "$POLL_INTERVAL"; ELAPSED=$(( ELAPSED + POLL_INTERVAL )); continue
  fi
  CONSECUTIVE_HTTP_FAILURES=0

  STATUS=$(echo "$BODY" | python3 -c "import sys,json; print(json.load(sys.stdin).get('result','UNKNOWN'))")
  echo "[INFO] Status: ${STATUS} (elapsed ${ELAPSED}s)"

  case "$STATUS" in
    PASSED|FAILED|FAILURE|STOPPED|ERROR|ABORTED)
      echo "[DONE] Terminal status reached: ${STATUS}"
      exit 0 ;;
  esac

  # Fail-safe: Testsigma's result vocabulary isn't fully documented, and it has
  # already been observed to diverge from what this list assumed (FAILURE vs
  # FAILED). If the same unrecognized, non-"in-progress" value shows up on two
  # consecutive polls, treat it as terminal and fail safe instead of polling
  # for hours (or, unbounded, forever) on a status this script doesn't know.
  STATUS_LC=$(echo "$STATUS" | tr '[:upper:]' '[:lower:]')
  if [[ -n "$STATUS" && "$IN_PROGRESS_STATUSES" != *" ${STATUS_LC} "* ]]; then
    if [[ "$STATUS_LC" == "$LAST_UNRECOGNIZED_LC" ]]; then
      echo "[WARN] Unrecognized status '${STATUS}' seen on two consecutive polls -- treating as terminal (failing safe) rather than waiting for a timeout. Please report this status value so it can be added to the known list."
      exit 1
    fi
    LAST_UNRECOGNIZED_LC="$STATUS_LC"
  else
    LAST_UNRECOGNIZED_LC=""
  fi

  sleep "$POLL_INTERVAL"
  ELAPSED=$(( ELAPSED + POLL_INTERVAL ))
done

# poll-status.ps1 -- see poll-status.sh header for parameter docs.
# Even when the normal cap is disabled ("unbounded"/"none"/0), an
# absolute ceiling (ABSOLUTE_MAX_WAIT_MINUTES, default 1440 = 24h)
# still applies. Also fails safe if Testsigma returns a result value
# outside the known terminal list on two consecutive polls.
# -------------------------------------------------------------
$ErrorActionPreference = "Continue"

$runIdFile = $env:RUN_ID_FILE
if (-not $env:TS_API_TOKEN) { Write-Host "[ERROR] TS_API_TOKEN is not set."; exit 1 }
if (-not (Test-Path $runIdFile)) { Write-Host "[ERROR] Run ID file not found at $runIdFile."; exit 1 }
$runId = (Get-Content $runIdFile -Raw).Trim()

$rawMaxWait = if ($env:MAX_WAIT_MINUTES) { $env:MAX_WAIT_MINUTES } else { "240" }
$rawMaxWaitLc = $rawMaxWait.ToLower()
$absoluteMaxMinutes = if ($env:ABSOLUTE_MAX_WAIT_MINUTES) { [int]$env:ABSOLUTE_MAX_WAIT_MINUTES } else { 1440 }
if ($rawMaxWaitLc -in @("0", "unbounded", "none", "")) {
  $capMode = "absolute-ceiling"; $maxSeconds = $absoluteMaxMinutes * 60; $displayMaxMinutes = $absoluteMaxMinutes
  Write-Host "[INFO] MAX_WAIT_MINUTES='$rawMaxWait' -> normal safety cap disabled. Polling until terminal status, backstopped by an absolute ceiling of $absoluteMaxMinutes minute(s) (override via ABSOLUTE_MAX_WAIT_MINUTES)."
} else {
  if ($rawMaxWait -notmatch '^[0-9]+$') { Write-Host "[ERROR] MAX_WAIT_MINUTES must be an integer, 'unbounded', or 'none'."; exit 1 }
  $capMode = "normal"; $maxSeconds = [int]$rawMaxWait * 60; $displayMaxMinutes = $rawMaxWait
  Write-Host "[INFO] MAX_WAIT_MINUTES='$rawMaxWait' -> safety cap ENABLED at ${maxSeconds}s."
}

# Statuses treated as "still in progress" -- excluded from the
# unrecognized-status fail-safe below.
$inProgressStatuses = @("running","pending","queued","in_progress","inprogress","scheduled","initiated","started","waiting","not_started","notstarted")

$pollInterval = if ($env:POLL_INTERVAL_SECONDS) { [int]$env:POLL_INTERVAL_SECONDS } else { 30 }
$maxHttpFailures = if ($env:MAX_CONSECUTIVE_HTTP_FAILURES) { [int]$env:MAX_CONSECUTIVE_HTTP_FAILURES } else { 10 }
$elapsed = 0
$consecutiveFailures = 0
$lastUnrecognizedLc = ""
Write-Host "[INFO] Polling Testsigma for Run ID: $runId"

while ($true) {
  if ($elapsed -ge $maxSeconds) {
    if ($capMode -eq "absolute-ceiling") {
      Write-Host "[TIMEOUT] Absolute ceiling of $displayMaxMinutes minute(s) reached (MAX_WAIT_MINUTES was set to '$rawMaxWait'). Failing the build."
    } else {
      Write-Host "[TIMEOUT] Safety cap of $displayMaxMinutes minute(s) reached. Failing the build."
    }
    exit 1
  }
  try {
    $resp = Invoke-RestMethod -Uri "https://app.testsigma.com/api/v1/execution_results/$runId" -Method Get `
      -Headers @{ "Authorization" = "Bearer $env:TS_API_TOKEN"; "Accept" = "application/json" }
    $consecutiveFailures = 0
    $status = $resp.result
    Write-Host "[INFO] Status: $status (elapsed ${elapsed}s)"
    if ($status -in @("PASSED","FAILED","FAILURE","STOPPED","ERROR","ABORTED")) {
      Write-Host "[DONE] Terminal status reached: $status"; exit 0
    }
    # Fail-safe: same unrecognized non-"in-progress" value on two
    # consecutive polls is treated as terminal rather than waited out.
    $statusLc = if ($status) { $status.ToString().ToLower() } else { "" }
    if ($statusLc -and ($inProgressStatuses -notcontains $statusLc)) {
      if ($statusLc -eq $lastUnrecognizedLc) {
        Write-Host "[WARN] Unrecognized status '$status' seen on two consecutive polls -- treating as terminal (failing safe) rather than waiting for a timeout. Please report this status value so it can be added to the known list."
        exit 1
      }
      $lastUnrecognizedLc = $statusLc
    } else {
      $lastUnrecognizedLc = ""
    }
  } catch {
    $consecutiveFailures++
    Write-Host "[WARN] Poll failed ($consecutiveFailures/$maxHttpFailures): $($_.Exception.Message)"
    if ($consecutiveFailures -ge $maxHttpFailures) {
      Write-Host "[ERROR] Testsigma API unreachable $maxHttpFailures polls in a row. Failing the build."; exit 1
    }
  }
  Start-Sleep -Seconds $pollInterval
  $elapsed += $pollInterval
}

The 4-Hour Failsafe

MAX_WAIT_MINUTES is a safety net, not an intended run length. If Testsigma never reports a finished result, the build fails after this many minutes instead of hanging forever. Default is 240 (4 hours). Set it to 0, unbounded, or none to disable it and wait indefinitely.

#!/bin/bash
# -------------------------------------------------------------
# poll-status.sh
# Polls Testsigma until the run reaches a terminal status, or
# (optionally) a safety-cap timeout elapses -- whichever comes
# first. Toggle the cap via MAX_WAIT_MINUTES without editing this file.
# Even when the cap is disabled ("unbounded"/"none"/0), an absolute
# ceiling (ABSOLUTE_MAX_WAIT_MINUTES, default 1440 = 24h) still applies,
# so this script can never loop truly forever. It also fails safe if
# Testsigma ever returns a result value outside the known terminal
# list on two consecutive polls, rather than hanging on an unrecognized
# status until a timeout eventually fires.
#
# Required: TS_API_TOKEN, RUN_ID_FILE
# Optional: MAX_WAIT_MINUTES (default: 240 = 4h; 0/"unbounded"/"none" disables
#             the normal cap in favor of the absolute ceiling below),
#           ABSOLUTE_MAX_WAIT_MINUTES (default: 1440 = 24h; last-resort
#             ceiling that applies even when MAX_WAIT_MINUTES is unbounded),
#           POLL_INTERVAL_SECONDS (default: 30),
#           MAX_CONSECUTIVE_HTTP_FAILURES (default: 10)
# -------------------------------------------------------------

set -uo pipefail
# NOTE: 'set -e' intentionally omitted -- one failed curl inside the loop
# must not kill the whole script; every HTTP call is checked explicitly.

RUN_ID_FILE="${RUN_ID_FILE:-/tmp/ts_run_id_manual-run}"
POLL_INTERVAL="${POLL_INTERVAL_SECONDS:-30}"
MAX_HTTP_FAILURES="${MAX_CONSECUTIVE_HTTP_FAILURES:-10}"
ABSOLUTE_MAX_MINUTES="${ABSOLUTE_MAX_WAIT_MINUTES:-1440}"

if [[ -z "${TS_API_TOKEN:-}" ]]; then
  echo "[ERROR] TS_API_TOKEN is not set."; exit 1
fi
if [[ ! -f "$RUN_ID_FILE" ]]; then
  echo "[ERROR] Run ID file not found at ${RUN_ID_FILE}. Did trigger-run.sh complete successfully?"; exit 1
fi
RUN_ID=$(cat "$RUN_ID_FILE")

RAW_MAX_WAIT="${MAX_WAIT_MINUTES:-240}"
RAW_MAX_WAIT_LC=$(echo "$RAW_MAX_WAIT" | tr '[:upper:]' '[:lower:]')
if [[ -z "$RAW_MAX_WAIT" || "$RAW_MAX_WAIT_LC" == "0" || "$RAW_MAX_WAIT_LC" == "unbounded" || "$RAW_MAX_WAIT_LC" == "none" ]]; then
  CAP_MODE="absolute-ceiling"; MAX_SECONDS=$(( ABSOLUTE_MAX_MINUTES * 60 )); DISPLAY_MAX_MINUTES="$ABSOLUTE_MAX_MINUTES"
  echo "[INFO] MAX_WAIT_MINUTES='${RAW_MAX_WAIT}' -> normal safety cap disabled. Polling until terminal status, backstopped by an absolute ceiling of ${ABSOLUTE_MAX_MINUTES} minute(s) (override via ABSOLUTE_MAX_WAIT_MINUTES)."
else
  if ! [[ "$RAW_MAX_WAIT" =~ ^[0-9]+$ ]]; then
    echo "[ERROR] MAX_WAIT_MINUTES must be a non-negative integer, 'unbounded', or 'none'. Got: '${RAW_MAX_WAIT}'"; exit 1
  fi
  CAP_MODE="normal"; MAX_SECONDS=$(( RAW_MAX_WAIT * 60 )); DISPLAY_MAX_MINUTES="$RAW_MAX_WAIT"
  echo "[INFO] MAX_WAIT_MINUTES='${RAW_MAX_WAIT}' -> safety cap ENABLED at ${MAX_SECONDS}s."
fi

# Statuses treated as "still in progress" -- never counted toward the
# unrecognized-status fail-safe below. UNKNOWN is this script's own
# placeholder when Testsigma's response has no 'result' field at all.
IN_PROGRESS_STATUSES=" running pending queued in_progress inprogress scheduled initiated started waiting not_started notstarted unknown "

ELAPSED=0
CONSECUTIVE_HTTP_FAILURES=0
LAST_UNRECOGNIZED_LC=""
echo "[INFO] Polling Testsigma for Run ID: ${RUN_ID}"

while true; do
  if [[ "$ELAPSED" -ge "$MAX_SECONDS" ]]; then
    if [[ "$CAP_MODE" == "absolute-ceiling" ]]; then
      echo "[TIMEOUT] Absolute ceiling of ${DISPLAY_MAX_MINUTES} minute(s) reached (MAX_WAIT_MINUTES was set to '${RAW_MAX_WAIT}'). Marking build FAILED."
    else
      echo "[TIMEOUT] Safety cap of ${DISPLAY_MAX_MINUTES} minute(s) reached. Marking build FAILED."
    fi
    echo "[TIMEOUT] Testsigma Run ID ${RUN_ID} is left RUNNING -- this script does not stop it on timeout, by design."
    exit 1
  fi

  TMPFILE=$(mktemp)
  HTTP_CODE=$(curl -s -o "$TMPFILE" -w "%{http_code}" \
    -X GET -H "Authorization: Bearer ${TS_API_TOKEN}" -H "Accept: application/json" \
    "https://app.testsigma.com/api/v1/execution_results/${RUN_ID}")
  BODY=$(cat "$TMPFILE"); rm -f "$TMPFILE"

  if [[ "$HTTP_CODE" != "200" ]]; then
    CONSECUTIVE_HTTP_FAILURES=$(( CONSECUTIVE_HTTP_FAILURES + 1 ))
    echo "[WARN] Poll failed HTTP ${HTTP_CODE} (${CONSECUTIVE_HTTP_FAILURES}/${MAX_HTTP_FAILURES}). Body: ${BODY}"
    if [[ "$CONSECUTIVE_HTTP_FAILURES" -ge "$MAX_HTTP_FAILURES" ]]; then
      echo "[ERROR] Testsigma API unreachable ${MAX_HTTP_FAILURES} polls in a row. Failing the build."; exit 1
    fi
    sleep "$POLL_INTERVAL"; ELAPSED=$(( ELAPSED + POLL_INTERVAL )); continue
  fi
  CONSECUTIVE_HTTP_FAILURES=0

  STATUS=$(echo "$BODY" | python3 -c "import sys,json; print(json.load(sys.stdin).get('result','UNKNOWN'))")
  echo "[INFO] Status: ${STATUS} (elapsed ${ELAPSED}s)"

  case "$STATUS" in
    PASSED|FAILED|FAILURE|STOPPED|ERROR|ABORTED)
      echo "[DONE] Terminal status reached: ${STATUS}"
      exit 0 ;;
  esac

  # Fail-safe: Testsigma's result vocabulary isn't fully documented, and it has
  # already been observed to diverge from what this list assumed (FAILURE vs
  # FAILED). If the same unrecognized, non-"in-progress" value shows up on two
  # consecutive polls, treat it as terminal and fail safe instead of polling
  # for hours (or, unbounded, forever) on a status this script doesn't know.
  STATUS_LC=$(echo "$STATUS" | tr '[:upper:]' '[:lower:]')
  if [[ -n "$STATUS" && "$IN_PROGRESS_STATUSES" != *" ${STATUS_LC} "* ]]; then
    if [[ "$STATUS_LC" == "$LAST_UNRECOGNIZED_LC" ]]; then
      echo "[WARN] Unrecognized status '${STATUS}' seen on two consecutive polls -- treating as terminal (failing safe) rather than waiting for a timeout. Please report this status value so it can be added to the known list."
      exit 1
    fi
    LAST_UNRECOGNIZED_LC="$STATUS_LC"
  else
    LAST_UNRECOGNIZED_LC=""
  fi

  sleep "$POLL_INTERVAL"
  ELAPSED=$(( ELAPSED + POLL_INTERVAL ))
done

    Write-Host "[WARN] Poll failed ($consecutiveFailures/$maxHttpFailures): $($_.Exception.Message)"
    if ($consecutiveFailures -ge $maxHttpFailures) {
      Write-Host "[ERROR] Testsigma API unreachable $maxHttpFailures polls in a row. Failing the build."; exit 1
    }
  }
  Start-Sleep -Seconds $pollInterval
  $elapsed += $pollInterval
}

# -------------------------------------------------------------
# fetch-report.sh
# Downloads raw JSON results for a finished Testsigma run into
# REPORT_DIR. SUITE and CASE auto-discover their own IDs via
# Testsigma's paginated list endpoints (TS_SUITE_RESULT_IDS /
# TS_CASE_RESULT_IDS can override with a manual comma-separated
# subset). STEP reuses the CASE id list. A short pace delay runs
# between requests to stay well clear of rate limits.
#
# Required: TS_API_TOKEN, RUN_ID_FILE, REPORT_DIR
# Optional: TS_REPORT_LEVELS (default "SUITE,CASE,STEP"; PLAN is
#           always fetched as a baseline), TS_SUITE_RESULT_IDS,
#           TS_CASE_RESULT_IDS, PACING_SECONDS (default 1)
# -------------------------------------------------------------

set -uo pipefail
RUN_ID_FILE="${RUN_ID_FILE:-/tmp/ts_run_id_manual-run}"
PACING="${PACING_SECONDS:-1}"
LEVELS_RAW="${TS_REPORT_LEVELS:-SUITE,CASE,STEP}"
if [[ "$LEVELS_RAW" == "ALL" ]]; then LEVELS_RAW="PLAN,SUITE,CASE,STEP"; fi
IFS=',' read -ra LEVELS <<< "$LEVELS_RAW"

if [[ -z "${TS_API_TOKEN:-}" ]]; then echo "[ERROR] TS_API_TOKEN is not set."; exit 1; fi
if [[ ! -f "$RUN_ID_FILE" ]]; then echo "[ERROR] Run ID file not found at ${RUN_ID_FILE}."; exit 1; fi
RUN_ID=$(cat "$RUN_ID_FILE")
REPORT_DIR="${REPORT_DIR:-./ts-reports}"
mkdir -p "$REPORT_DIR"
FAILURES=0

ts_get() {
  # $1 url, $2 out file -- returns 0 ok, 1 http failure
  local code
  code=$(curl -s -o "$2" -w "%{http_code}" -H "Authorization: Bearer ${TS_API_TOKEN}" -H "Accept: application/json" "$1")
  if [[ "$code" != "200" ]]; then echo "[ERROR] GET ${1} -> HTTP ${code}" >&2; return 1; fi
  return 0
}

ts_get_paginated_ids() {
  # $1 base_url, $2 out_prefix (writes each item as <prefix><id>.json), $3 label
  # prints discovered ids, one per line, to stdout
  local base_url="$1" out_prefix="$2" label="$3" page=0 total_pages=1
  while [[ "$page" -lt "$total_pages" ]]; do
    local paged; paged=$(mktemp)
    if ! ts_get "${base_url}?page=${page}&size=100" "$paged"; then rm -f "$paged"; return 1; fi
    python3 -c "
import json
with open('$paged') as f: data = json.load(f)
for item in data.get('content', []):
    iid = item.get('id')
    if iid is not None:
        with open('${out_prefix}' + str(iid) + '.json', 'w') as out:
            json.dump(item, out)
        print(iid)
print('__TOTAL_PAGES__' + str(data.get('totalPages', 1) or 1), file=__import__('sys').stderr)
" > "${paged}.ids" 2>"${paged}.meta" || { echo "[ERROR] Failed to parse ${label} list (page ${page})."; rm -f "$paged" "${paged}.ids" "${paged}.meta"; return 1; }
    total_pages=$(grep -o '__TOTAL_PAGES__[0-9]*' "${paged}.meta" | grep -o '[0-9]*$')
    cat "${paged}.ids"
    rm -f "$paged" "${paged}.ids" "${paged}.meta"
    page=$(( page + 1 ))
    sleep "$PACING"
  done
}

# --- PLAN (always fetched as a baseline) ---
echo "[INFO] Fetching PLAN level..."
ts_get "https://app.testsigma.com/api/v2/test_runs/${RUN_ID}" "${REPORT_DIR}/${RUN_ID}_plan.json" || FAILURES=$(( FAILURES + 1 ))
sleep "$PACING"

# --- SUITE ---
if [[ " ${LEVELS[*]} " == *" SUITE "* ]]; then

    ids=$(ts_get_paginated_ids "https://app.testsigma.com/api/v2/test_runs/${RUN_ID}/test_suite_results" "${REPORT_DIR}/${RUN_ID}_suite_" "SUITE") || FAILURES=$(( FAILURES + 1 ))
    echo "[OK] SUITE -> $(echo "$ids" | grep -c . || true) result(s) auto-discovered"
  fi
fi

# --- CASE (ids shared with STEP) ---
CASE_IDS=""
if [[ " ${LEVELS[*]} " == *" CASE "* || " ${LEVELS[*]} " == *" STEP "* ]]; then
  if [[ -n "${TS_CASE_RESULT_IDS:-}" ]]; then
    CASE_IDS="${TS_CASE_RESULT_IDS// /}"
    IFS=',' read -ra CIDS <<< "$CASE_IDS"
    if [[ " ${LEVELS[*]} " == *" CASE "* ]]; then
      for cid in "${CIDS[@]}"; do
        echo "[INFO] Fetching CASE ${cid} (manual override)..."

  else

    CASE_IDS=$(ts_get_paginated_ids "https://app.testsigma.com/api/v2/test_runs/${RUN_ID}/test_case_results" "${REPORT_DIR}/${RUN_ID}_case_" "CASE" | tr '\n' ',') || FAILURES=$(( FAILURES + 1 ))
    if [[ " ${LEVELS[*]} " == *" CASE "* ]]; then
      echo "[OK] CASE -> auto-discovered"
    fi
  fi
fi

  echo "[INFO] Fetching STEP level for each case result..."
  IFS=',' read -ra CIDS <<< "${CASE_IDS%,}"
  for cid in "${CIDS[@]}"; do

    sleep "$PACING"
  done
fi

echo "[INFO] fetch-report.sh complete with ${FAILURES} failure(s)."
exit $(( FAILURES > 0 ? 1 : 0 ))

#!/usr/bin/env python3
# -------------------------------------------------------------
# generate-html-report.py
# Reads every *_plan.json / *_suite_*.json / *_case_*.json /
# *_step_*.json in REPORT_DIR and renders one self-contained,
# offline-readable HTML file: ts-report.html in the same folder.
# Plan Summary leads in plain English; everything else sits one
# click away behind native <details> elements -- no JS framework,
# no external assets, works the same offline as everything else
# in this pipeline.
# -------------------------------------------------------------
import json, os, glob, html, sys

report_dir = os.environ.get("REPORT_DIR", "./ts-reports")

def load(pattern):
    out = []
    for path in sorted(glob.glob(os.path.join(report_dir, pattern))):
        try:
            with open(path) as f:
                out.append((path, json.load(f)))
        except (json.JSONDecodeError, OSError) as e:
            print(f"[WARN] Could not read {path}: {e}", file=sys.stderr)
    return out

plan_files  = load("*_plan.json")
suite_files = load("*_suite_*.json")
case_files  = load("*_case_*.json")
step_files  = load("*_step_*.json")

def esc(v):
    return html.escape(str(v)) if v is not None else ""

plan = plan_files[0][1] if plan_files else {}
summary_rows = "".join(
    f"<tr><td>{esc(k)}</td><td>{esc(plan.get(k))}</td></tr>"
    for k in ["result","duration","startTime","endTime","triggeredBy","buildNo",
              "totalPassedCount","totalFailedCount","totalSkippedCount"] if k in plan
)

def table(files, cols, label):
    if not files:
        return f"<p>No {label} results fetched.</p>"
    head = "".join(f"<th>{esc(c)}</th>" for c in cols)
    rows = []
    for _, item in files:
        cells = "".join(f"<td>{esc(item.get(c))}</td>" for c in cols)
        rows.append(f"<tr>{cells}</tr>")
    return f"<table><thead><tr>{head}</tr></thead><tbody>{''.join(rows)}</tbody></table>"

suite_html = table(suite_files, ["id","name","result","duration"], "SUITE")
case_html  = table(case_files,  ["id","name","result","duration"], "CASE")

step_sections = []
for path, payload in step_files:
    case_id = os.path.basename(path).split("_step_")[-1].replace(".json", "")
    steps = payload.get("content", payload if isinstance(payload, list) else [])
    rows = "".join(
        f"<tr><td>{esc(s.get('id'))}</td><td>{esc(s.get('name') or s.get('description'))}</td><td>{esc(s.get('result'))}</td></tr>"
        for s in steps if isinstance(s, dict)
    )
    step_sections.append(f"<h4>Case {esc(case_id)}</h4><table><thead><tr><th>Step ID</th><th>Name</th><th>Result</th></tr></thead><tbody>{rows}</tbody></table>")

            echo "[ABORT] WARNING: Testsigma did not confirm the stop (see HTTP error above). The run may still be RUNNING and consuming a license seat -- check Testsigma UI."
          }
        } else {
          echo "[ABORT] No Run ID found -- Testsigma may not have started."
        }
        fetchAndGenerateReport()
      }
    }
    failure {
      script { echo "[FAILED] Check console output above."; fetchAndGenerateReport() }
    }
    success {
      script { echo "[SUCCESS] Testsigma run completed and passed."; fetchAndGenerateReport() }
    }
  }
}

Reading Your HTML Report

ts-report.html opens in any browser — no internet connection needed once it's downloaded. It leads with a Plan Summary: result, duration, start/end time, who triggered it, and pass/fail counts. Suite and Case tables follow, then Step-level detail grouped by case. The full raw data sits behind a "Raw plan JSON" toggle so it doesn't get in the way.

Reference — Endpoints & Key Decisions

Why is fetch/report called from aborted/failure/success separately, instead of a single always {} block? Jenkins always runs always {} before aborted/failure/success, regardless of the order they're written in the file. If fetch/report lived in always {}, an aborted build's report would be generated before the Testsigma stop signal goes out — capturing a stale "RUNNING" snapshot instead of the final "STOPPED" result. Calling the shared function explicitly from each branch, after the abort handling, avoids that race.

Why sh(..., returnStatus: true) for fetch/report instead of letting them fail the build directly? A failure in report generation isn't the same kind of failure as the Test Plan itself failing. Capturing the exit status and logging a [WARN] means a report-generation hiccup doesn't overwrite the real pass/fail outcome that the build's status should reflect.

Why does RUN_ID_FILE live in a fixed path outside the workspace instead of inside it? The abort handler needs to find the Run ID no matter what state the workspace is in when Abort is clicked — a fixed path in /tmp (Mac) or %TEMP% (Windows) survives that reliably.
