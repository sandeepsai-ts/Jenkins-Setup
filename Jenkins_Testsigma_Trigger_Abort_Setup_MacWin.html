Before You Start

You need: a Mac or Windows machine with admin access, a Testsigma account with at least one Test Plan already set up, and 30–45 uninterrupted minutes. You do not need to already know Groovy, Jenkins, or shell/PowerShell scripting — just follow the steps in order for whichever OS you're on (use the toggle above).

This builds a job that runs automatically every night, starts a Testsigma test, watches it until it finishes, and — if you ever hit Abort in Jenkins — tells Testsigma to stop too, so it doesn't keep running (and using up your test license) in the background.

Nothing in these scripts or in the Jenkinsfile is tied to one person's machine. Every path is built from environment variables that Jenkins fills in automatically ($(whoami) / $env:USERNAME, env.HOME / env.USERPROFILE, env.JOB_NAME), so this guide works as-is on any Mac or Windows agent. Anywhere you'd normally have to hunt down and edit a hardcoded value, you'll instead see a 🔧 Customizable flag telling you exactly what it is and where it lives.

Part 3 — One Thing to Know Before Continuing

Jenkins has an official Testsigma plugin. This guide does not use it, on purpose — it has an unresolved security issue (Jenkins Security Advisory 2025-07-09, CVE-2025-53661: it doesn't hide your Testsigma API key properly) and hasn't been updated since 2022. This setup uses plain scripts (shell on Mac, PowerShell on Windows) and Jenkins' built-in Pipeline feature instead — nothing Testsigma-specific to install here, on either OS.

Part 5 — Create the Scripts Folder

This job calls two small helper scripts. Create a folder and paste in the code.

17

Open a plain text editor (TextEdit → Format → Make Plain Text). Paste the following exactly, save as trigger-run.sh inside the folder from Step 16:

#!/bin/bash
# -------------------------------------------------------------
# trigger-run.sh
# Triggers a Testsigma execution and saves the Run ID to a
# fixed path in /tmp so the Jenkins abort hook can always find
# it regardless of workspace state.
#
# Required environment variables (injected by Jenkins):
#   TS_API_TOKEN  -- Testsigma API key (from Jenkins Credential TS_API_KEY)
#   TS_PLAN_ID    -- Execution ID from Testsigma CI/CD Integrations tab (from Jenkins Credential TS_PLAN_ID)
#   APP_VERSION   -- passed as buildNo for traceability in Testsigma UI
#   RUN_ID_FILE   -- fixed path set in the Jenkinsfile environment block
# -------------------------------------------------------------

set -euo pipefail

if [[ -z "${TS_PLAN_ID:-}" || -z "${TS_API_TOKEN:-}" ]]; then
  echo "[ERROR] TS_PLAN_ID and TS_API_TOKEN must be set."
  exit 1
fi

echo "[INFO] Triggering plan ID: ${TS_PLAN_ID} | Version: ${APP_VERSION:-unset}"

TMPFILE=$(mktemp)
HTTP_CODE=$(curl -s -o "$TMPFILE" -w "%{http_code}" \
  -X POST \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -H "Authorization: Bearer ${TS_API_TOKEN}" \
  "https://app.testsigma.com/api/v1/execution_results" \
  -d "{\"executionId\": \"${TS_PLAN_ID}\", \"buildNo\": \"${APP_VERSION:-manual}\"}")

BODY=$(cat "$TMPFILE")
rm -f "$TMPFILE"

if [[ "$HTTP_CODE" != "200" && "$HTTP_CODE" != "201" ]]; then
  echo "[ERROR] Testsigma returned HTTP ${HTTP_CODE}"
  echo "$BODY"
  exit 1
fi

RUN_ID=$(echo "$BODY" | python3 -c "import sys,json; print(json.load(sys.stdin)['id'])")

if [[ -z "$RUN_ID" || "$RUN_ID" == "null" ]]; then
  echo "[ERROR] Could not read Run ID from response."
  echo "$BODY"
  exit 1
fi

# Write to the fixed path declared in the Jenkinsfile -- survives
# workspace cleanup and Jenkins abort. The fallback below only
# applies if you run this script manually, outside Jenkins.
echo "$RUN_ID" > "${RUN_ID_FILE:-/tmp/ts_run_id_manual-run}"
echo "[OK] Execution started. Run ID: ${RUN_ID}"
echo "[INFO] Run ID saved to: ${RUN_ID_FILE:-/tmp/ts_run_id_manual-run}"

Repeat for a second file, poll-status.sh, same folder:

#!/bin/bash
# -------------------------------------------------------------
# poll-status.sh
# Polls Testsigma every 30 seconds until the run completes,
# times out, or is interrupted by a Jenkins abort.
#
# Required environment variables (injected by Jenkins):
#   TS_API_TOKEN      -- Testsigma API key
#   MAX_WAIT_MINUTES  -- timeout in minutes (set per plan in Jenkinsfile)
#   RUN_ID_FILE       -- fixed path set in the Jenkinsfile environment block
# -------------------------------------------------------------

set -euo pipefail

RUN_ID_FILE="${RUN_ID_FILE:-/tmp/ts_run_id_manual-run}"

if [[ -z "${TS_API_TOKEN:-}" ]]; then
  echo "[ERROR] TS_API_TOKEN is not set."
  exit 1
fi

if [[ ! -f "$RUN_ID_FILE" ]]; then
  echo "[ERROR] Run ID file not found at ${RUN_ID_FILE}."
  echo "[ERROR] Did trigger-run.sh complete successfully?"
  exit 1
fi

RUN_ID=$(cat "$RUN_ID_FILE")
MAX_WAIT="${MAX_WAIT_MINUTES:-30}"
POLL_INTERVAL=30
MAX_SECONDS=$(( MAX_WAIT * 60 ))
ELAPSED=0

echo "[INFO] Polling Testsigma for Run ID: ${RUN_ID}"

while true; do
  if (( ELAPSED >= MAX_SECONDS )); then
    echo "[TIMEOUT] Marking Jenkins build as FAILED."
    exit 1
  fi

  TMPFILE=$(mktemp)
  HTTP_CODE=$(curl -s -o "$TMPFILE" -w "%{http_code}" \
    -X GET \
    -H "Authorization: Bearer ${TS_API_TOKEN}" \
    -H "Accept: application/json" \
    "https://app.testsigma.com/api/v1/execution_results/${RUN_ID}")

  BODY=$(cat "$TMPFILE"); rm -f "$TMPFILE"

  if [[ "$HTTP_CODE" != "200" ]]; then
    sleep "$POLL_INTERVAL"; ELAPSED=$(( ELAPSED + POLL_INTERVAL )); continue
  fi

  RESULT=$(echo "$BODY" | python3 -c "import sys,json; print(json.load(sys.stdin).get('result','UNKNOWN'))")
  STATUS=$(echo "$BODY" | python3 -c "import sys,json; print(json.load(sys.stdin).get('status','UNKNOWN'))")

  echo "Run ID: ${RUN_ID} | result: ${RESULT} | status: ${STATUS}"

  case "$RESULT" in
    PASSED)          echo "[SUCCESS] PASSED.";             exit 0 ;;
    FAILURE|FAILED)  echo "[FAILED] Run FAILED.";        exit 1 ;;
    STOPPED|ABORTED) echo "[STOPPED] Run stopped/aborted."; exit 1 ;;
    *)               sleep "$POLL_INTERVAL"; ELAPSED=$(( ELAPSED + POLL_INTERVAL )) ;;
  esac
done

# -------------------------------------------------------------
# trigger-run.ps1
# Triggers a Testsigma execution and saves the Run ID to a fixed
# path so the Jenkins abort hook can always find it regardless
# of workspace state.
#
# Required environment variables (injected by Jenkins):
#   TS_API_TOKEN  - Testsigma API key (from Jenkins Credential TS_API_KEY)
#   TS_PLAN_ID    - Execution ID from Testsigma CI/CD Integrations tab (from Jenkins Credential TS_PLAN_ID)
#   APP_VERSION   - passed as buildNo for traceability in Testsigma UI
#   RUN_ID_FILE   - fixed path set in the Jenkinsfile environment block
# -------------------------------------------------------------

$ErrorActionPreference = "Stop"

if ([string]::IsNullOrWhiteSpace($env:TS_PLAN_ID) -or [string]::IsNullOrWhiteSpace($env:TS_API_TOKEN)) {
    Write-Host "[ERROR] TS_PLAN_ID and TS_API_TOKEN must be set."
    exit 1
}

$appVersion = if ($env:APP_VERSION) { $env:APP_VERSION } else { "unset" }
Write-Host "[INFO] Triggering plan ID: $($env:TS_PLAN_ID) | Version: $appVersion"

$runIdFile = if ($env:RUN_ID_FILE) { $env:RUN_ID_FILE } else { "$env:TEMP\ts_run_id_manual-run.txt" }
$buildNo   = if ($env:APP_VERSION) { $env:APP_VERSION } else { "manual" }

try {
    $body = @{ executionId = $env:TS_PLAN_ID; buildNo = $buildNo } | ConvertTo-Json
    $response = Invoke-RestMethod -Uri "https://app.testsigma.com/api/v1/execution_results" `
        -Method Post `
        -Headers @{ "Authorization" = "Bearer $($env:TS_API_TOKEN)"; "Accept" = "application/json" } `
        -ContentType "application/json" `
        -Body $body
}
catch {
    Write-Host "[ERROR] Testsigma request failed: $($_.Exception.Message)"
    if ($_.ErrorDetails.Message) { Write-Host $_.ErrorDetails.Message }
    exit 1
}

$runId = $response.id
if ([string]::IsNullOrWhiteSpace($runId) -or $runId -eq "null") {
    Write-Host "[ERROR] Could not read Run ID from response."
    Write-Host ($response | ConvertTo-Json)
    exit 1
}

Set-Content -Path $runIdFile -Value $runId -NoNewline
Write-Host "[OK] Execution started. Run ID: $runId"
Write-Host "[INFO] Run ID saved to: $runIdFile"

Repeat for a second file, poll-status.ps1, same folder:

# -------------------------------------------------------------
# poll-status.ps1
# Polls Testsigma every 30 seconds until the run completes,
# times out, or is interrupted by a Jenkins abort.
#
# Required environment variables (injected by Jenkins):
#   TS_API_TOKEN      - Testsigma API key
#   MAX_WAIT_MINUTES  - timeout in minutes (set per plan in Jenkinsfile)
#   RUN_ID_FILE       - fixed path set in the Jenkinsfile environment block
# -------------------------------------------------------------

$ErrorActionPreference = "Stop"

$runIdFile = if ($env:RUN_ID_FILE) { $env:RUN_ID_FILE } else { "$env:TEMP\ts_run_id_manual-run.txt" }

if ([string]::IsNullOrWhiteSpace($env:TS_API_TOKEN)) {
    Write-Host "[ERROR] TS_API_TOKEN is not set."
    exit 1
}

if (-not (Test-Path $runIdFile)) {
    Write-Host "[ERROR] Run ID file not found at $runIdFile."
    Write-Host "[ERROR] Did trigger-run.ps1 complete successfully?"
    exit 1
}

$runId          = (Get-Content -Path $runIdFile -Raw).Trim()
$maxWaitMinutes = if ($env:MAX_WAIT_MINUTES) { [int]$env:MAX_WAIT_MINUTES } else { 30 }
$pollInterval   = 30
$maxSeconds     = $maxWaitMinutes * 60
$elapsed        = 0

Write-Host "[INFO] Polling Testsigma for Run ID: $runId"

while ($true) {
    if ($elapsed -ge $maxSeconds) {
        Write-Host "[TIMEOUT] Marking Jenkins build as FAILED."
        exit 1
    }

    try {
        $response = Invoke-RestMethod -Uri "https://app.testsigma.com/api/v1/execution_results/$runId" `
            -Method Get `
            -Headers @{ "Authorization" = "Bearer $($env:TS_API_TOKEN)"; "Accept" = "application/json" }
    }
    catch {
        Write-Host "[WARN] Poll request failed, retrying: $($_.Exception.Message)"
        Start-Sleep -Seconds $pollInterval
        $elapsed += $pollInterval
        continue
    }

    $result = if ($response.result) { $response.result } else { "UNKNOWN" }
    $status = if ($response.status) { $response.status } else { "UNKNOWN" }

    Write-Host "Run ID: $runId | result: $result | status: $status"

    switch -Regex ($result) {
        "PASSED"          { Write-Host "[SUCCESS] PASSED.";              exit 0 }
        "FAILURE|FAILED"  { Write-Host "[FAILED] Run FAILED.";         exit 1 }
        "STOPPED|ABORTED" { Write-Host "[STOPPED] Run stopped/aborted."; exit 1 }
        default           { Start-Sleep -Seconds $pollInterval; $elapsed += $pollInterval }
    }
}

// -------------------------------------------------------------
// Jenkinsfile -- Mac agent
// Last updated: July 2026
//
// Fixes applied:
//   1. always block no longer deletes the Run ID file before the
//      aborted block can read it -- cleanup moved to end of each
//      post condition individually
//   2. Replaced new java.io.File() with sh+readFile pattern to
//      avoid Jenkins sandbox security rejection
// -------------------------------------------------------------
pipeline {
  agent any
  triggers {
    cron('H 2 * * *')
  }
  environment {
    // [CUSTOMIZE] must match the Credential ID from Part 6, Step 23
    TS_API_TOKEN     = credentials('TS_API_KEY')
    TS_PLAN_ID       = credentials('TS_PLAN_ID')
    APP_VERSION      = "2.2.2.2"
    // env.HOME resolves to the agent's own home folder at runtime --
    // no hardcoded username. Change only if you saved the scripts
    // folder somewhere other than the default path from Part 5.
    SCRIPTS_DIR      = "${env.HOME}/jenkins-ts-scripts"
    // [AUTO] Derived automatically from this job's own name (env.JOB_NAME) --
    // renaming the job in Step 26 does not require touching this line.
    RUN_ID_FILE      = "/tmp/ts_run_id_${env.JOB_NAME}"
    // -- Change this per test plan -----------------------------
    MAX_WAIT_MINUTES = "30"
  }
  stages {
    stage('Trigger Testsigma Run') {
      steps { sh '"${SCRIPTS_DIR}/trigger-run.sh"' }
    }
    stage('Wait for Results') {
      steps { sh '"${SCRIPTS_DIR}/poll-status.sh"' }
    }
  }
  post {
    // Fires ONLY when Abort is clicked in Jenkins
    aborted {
      script {
        def runId = sh(
          script: 'cat "${RUN_ID_FILE}" 2>/dev/null || echo ""',
          returnStdout: true
        ).trim()
        if (runId) {
          echo "[ABORT] Jenkins cancelled. Stopping Testsigma Run ID: ${runId}"
          sh """
            curl -s -X PUT \\
              -H "Authorization: Bearer ${TS_API_TOKEN}" \\
              -H "Content-Type: application/json" \\
              -H "Accept: application/json" \\
              "https://app.testsigma.com/api/v1/execution_results/${runId}" \\
              -d '{"result":"STOPPED"}' || true
          """
          echo "[ABORT] Stop signal sent. License freed."
        } else {
          echo "[ABORT] No Run ID found -- Testsigma may not have started. Check manually."
        }
        sh 'rm -f "${RUN_ID_FILE}" || true'
        echo "[CLEANUP] Done."
      }
    }
    failure {
      script {
        echo "[FAILED] Pipeline failed. Check console output above for details."
        sh 'rm -f "${RUN_ID_FILE}" || true'
        echo "[CLEANUP] Done."
      }
    }
    success {
      script {
        echo "[SUCCESS] Testsigma run completed and passed."
        sh 'rm -f "${RUN_ID_FILE}" || true'
        echo "[CLEANUP] Done."
      }
    }
  }
}

// -------------------------------------------------------------
// Jenkinsfile -- Windows agent
// Last updated: July 2026
//
// Notes:
//   1. Uses the built-in `powershell` pipeline step (bundled with
//      the standard Pipeline plugin set -- already installed if you
//      chose "Install suggested plugins" in Part 2).
//   2. runId is passed into the PowerShell abort block via
//      withEnv() rather than Groovy string interpolation, so the
//      PowerShell script text can stay single-quoted and safe.
// -------------------------------------------------------------
pipeline {
  agent any
  triggers {
    cron('H 2 * * *')
  }
  environment {
    // [CUSTOMIZE] must match the Credential ID from Part 6, Step 23
    TS_API_TOKEN     = credentials('TS_API_KEY')
    TS_PLAN_ID       = credentials('TS_PLAN_ID')
    APP_VERSION      = "2.2.2.2"
    // env.USERPROFILE resolves to the agent's own profile folder at
    // runtime -- no hardcoded username. Change only if you saved the
    // scripts folder somewhere other than the default path from Part 5.
    SCRIPTS_DIR      = "${env.USERPROFILE}\\jenkins-ts-scripts"
    // [AUTO] Derived automatically from this job's own name (env.JOB_NAME) --
    // renaming the job in Step 26 does not require touching this line.
    RUN_ID_FILE      = "${env.TEMP}\\ts_run_id_${env.JOB_NAME}.txt"
    // -- Change this per test plan -----------------------------
    MAX_WAIT_MINUTES = "30"
  }
  stages {
    stage('Trigger Testsigma Run') {
      steps { powershell '& "$env:SCRIPTS_DIR\\trigger-run.ps1"' }
    }
    stage('Wait for Results') {
      steps { powershell '& "$env:SCRIPTS_DIR\\poll-status.ps1"' }
    }
  }
  post {
    // Fires ONLY when Abort is clicked in Jenkins
    aborted {
      script {
        def runId = powershell(
          script: 'if (Test-Path $env:RUN_ID_FILE) { (Get-Content $env:RUN_ID_FILE -Raw).Trim() } else { Write-Output "" }',
          returnStdout: true
        ).trim()
        if (runId) {
          echo "[ABORT] Jenkins cancelled. Stopping Testsigma Run ID: ${runId}"
          withEnv(["TS_RUN_ID=${runId}"]) {
            powershell '''
              try {
                Invoke-RestMethod -Uri "https://app.testsigma.com/api/v1/execution_results/$env:TS_RUN_ID" -Method Put -Headers @{ "Authorization" = "Bearer $env:TS_API_TOKEN"; "Accept" = "application/json" } -ContentType "application/json" -Body \'{"result":"STOPPED"}\' | Out-Null
              } catch {
                Write-Host "[WARN] Stop request failed: $($_.Exception.Message)"
              }
            '''
          }
          echo "[ABORT] Stop signal sent. License freed."
        } else {
          echo "[ABORT] No Run ID found -- Testsigma may not have started. Check manually."
        }
        powershell 'Remove-Item -Path $env:RUN_ID_FILE -ErrorAction SilentlyContinue'
        echo "[CLEANUP] Done."
      }
    }
    failure {
      script {
        echo "[FAILED] Pipeline failed. Check console output above for details."
        powershell 'Remove-Item -Path $env:RUN_ID_FILE -ErrorAction SilentlyContinue'
        echo "[CLEANUP] Done."
      }
    }
    success {
      script {
        echo "[SUCCESS] Testsigma run completed and passed."
        powershell 'Remove-Item -Path $env:RUN_ID_FILE -ErrorAction SilentlyContinue'
        echo "[CLEANUP] Done."
      }
    }
  }
}
If Jenkins reports "powershell step not found" — install the PowerShell plugin via Manage Jenkins → Plugins. It's normally pulled in by "Install suggested plugins" in Part 2, but some minimal installs skip it.

Reference — Why This Works the Way It Does

Why not the official Testsigma Jenkins plugin? It has an unfixed security issue (CVE-2025-53661 — doesn't hide your API key properly) and hasn't been updated since 2022. It also can't do the abort part — only trigger and wait. This applies equally to Mac and Windows installs, since it's the same plugin either way.

Why separate aborted/failure/success cleanup instead of one shared step? Jenkins runs these in a fixed order, and a shared "always" cleanup step would run before the abort logic gets to read the saved Run ID — deleting the file it needs. Cleaning up at the end of each individual block avoids that race.

Why sh(script: 'cat ...') instead of reading the file directly in Groovy? Jenkins pipeline scripts run in a security sandbox that blocks raw Groovy/Java file reads unless an administrator approves each one. Routing through a shell command sidesteps this since shell commands are already allowed.

Why powershell(script: 'Get-Content ...') instead of reading the file directly in Groovy? Same sandbox restriction as on Mac — raw Groovy/Java file reads need admin approval, so the read is routed through PowerShell instead, which pipeline steps are already allowed to call.

Why pass the Run ID into the abort PowerShell block via withEnv() instead of Groovy string interpolation? The stop-call script is wrapped in Groovy triple-single-quotes so PowerShell's own $env: syntax isn't accidentally interpreted by Groovy first. That means the Run ID can't be spliced in with ${runId} inside that block — so it's exported as an environment variable instead, which PowerShell then reads natively.
